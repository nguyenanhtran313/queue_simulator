# queue_simulator — ghi chú cho Claude Code

Repo này là một **sandbox cá nhân**, chứa nhiều dự án con độc lập. Không coi đây là một app duy nhất — mỗi thư mục có mục đích và cách chạy riêng.

## Cấu trúc thư mục

| Thư mục | Nội dung | Cách chạy |
|---|---|---|
| `Queue Simulator/` | **Universal Queue Simulator** — web app mô phỏng lý thuyết xếp hàng sân bay. `index.html` là single-file React (UMD) + Babel-standalone + Tailwind CDN + Chart.js, **không có build step**. `implementation_plan.md` là tài liệu thiết kế bổ sung, `_REQUIREMENT.md` là đề bài/kế hoạch triển khai gốc. `presentation.html`/`index_backup.html`/`index_final_backup.html` là các bản lưu/trình bày. | Mở thẳng `index.html` bằng trình duyệt. |
| `Test_VSF_driver_allocation/` | Mô phỏng phân bổ tài xế + điều phối xe (Smart Dispatching), là **Phase 3 & 4** của case study "Xanh SM" tuyển dụng Data Team — xem `_REQUIREMENT.md`. `phase1_simulation.py` chạy simulation, `build_presentation.py` đọc kết quả và sinh `presentation.html`. Dữ liệu mock: `mock_drivers.csv`, `mock_riders.csv`, `realtime_heatmap.csv`. **Đây là khu vực đang được sửa tích cực nhất.** | `python phase1_simulation.py` rồi `python build_presentation.py` |
| `Test_VSF_forecast_demand/` | Dự báo nhu cầu đặt xe theo lưới H3 + LightGBM — là **Phase 1 & 2** của cùng case study "Xanh SM" nói trên (xem `_REQUIREMENT.md` để rõ mối liên hệ với `Test_VSF_driver_allocation/`). `step1_2_pipeline.py` sinh mock data + train model, `generate_eda.py` vẽ thêm EDA, output vào `output/`, có `Vinsmart_Presentation.html`. ⚠️ 2 script đang hard-code đường dẫn output tuyệt đối tới máy khác — phải sửa path trước khi chạy lại. | `python step1_2_pipeline.py` rồi `python generate_eda.py` |
| `Test_VSF_dinhgia_bds/` | Case study định giá bất động sản tự động (AVM) — **không liên quan** tới 2 thư mục VSF ở trên dù trùng tiền tố tên, xem `_REQUIREMENT.md`. `generate_data.py` sinh `real_estate_valuation_dataset.csv`, `compare_models.py` so sánh 5 thuật toán regression (best: LightGBM, R²≈97.3%). | `python generate_data.py` rồi `python compare_models.py` |
| `Basic ML/` | Chuỗi bài tự học ML (case study dự đoán khách hàng ngân hàng phản hồi khuyến mãi, 100.000 dòng mock — xem `_REQUIREMENT.md`), **đánh số thứ tự và phải chạy tuần tự** vì bài sau dùng output (`.pkl`, `.csv`, `.json`) của bài trước: `00_generate_data` → `01_EDA_and_Stats` → `02_KMeans_Segmentation` → `03_Logistic_Regression_Benchmark` → `04_XGBoost_Production` **và** `04b_LightGBM_Production` (cả 2, không phải chọn 1) → `04c_Model_Comparison` (so sánh 2 model) → `05_SHAP_Analysis`/`05b_SHAP_Analysis_LightGBM` → `06_Expected_Profit_Calculation` → `07_Uplift_Modeling_ROI` **và** `07b_Uplift_Modeling_ROI_LightGBM` (S-learner XGBoost vs LightGBM, quyết định chiến lược targeting cuối cùng theo lợi nhuận nhân quả) → `build_presentation.py` (đọc toàn bộ artifact, ra `presentation.html` phong cách Google Analytics/Analytics Design System — xem skill `design-html-page`; có bộ chọn nhanh ở đầu trang để nhảy sang các tài liệu khác trong "họ" Basic DA/DE/Data Governance). ⚠️ Cũng tồn tại `08_Model_Comparison_ROI.py` + `09_Build_Presentation.py` (2026-07-03) — một hướng làm khác kỹ hơn (chỉ chia train/test 1 lần dùng chung cho cả 6 model để tránh data leakage mà 03/04/04b/06/07 đang mắc phải), nhưng **chưa phải luồng chính thức** đang chạy ở trên — chưa được gộp vào, cần quyết định có nên thay thế không trước khi coi là chuẩn. | `python 0X_Ten_Bai.py` theo đúng thứ tự, kết thúc bằng `python build_presentation.py` |
| `Basic DA/` | Cẩm nang phỏng vấn Data Analyst — `DA_Interview_Lecture.html`, tài liệu 1 trang viết tay (không phải generated file), tổng hợp kỹ thuật thống kê/ML từ EDA đến Causal Inference, rút trực tiếp từ pipeline `Basic ML/`. Theo phong cách Analytics Design System (skill `design-html-page`), có bộ chọn nhanh ở đầu trang để nhảy sang các tài liệu khác. | Mở thẳng file `.html` bằng trình duyệt. |
| `Basic DE/` | 3 trang kiến trúc dữ liệu tham khảo, viết tay: `onemount_architecture.html` (ELT của OneMount trên GCP: Kafka→Spark→GCS Bronze→Spark ETL→Iceberg→Dremio), `master_rise_aws_architecture.html` (kiến trúc tương ứng phía Masterise dùng Databricks trên nền lưu trữ AWS S3 — xem disclaimer đầu trang, một số chi tiết vẫn là giả định chưa xác nhận), `onemount_vs_masterise_comparison.html` (so sánh song song 2 kiến trúc trên, theo đúng quy tắc màu Segment 1/2 của ADS). Cả 3 có bộ chọn nhanh liên kết sang tài liệu khác. | Mở thẳng file `.html` tương ứng bằng trình duyệt. |
| `Data Governance/` | Tài liệu học Data Governance. `DG DAMA DMBOK2.md`, `DG Explained in 5 minutes.md`, `DG ITSM.md`, `DG in Banking.md`, `DG in World of AI & ML.md`, `Introduction to DG.md` là ghi chú học tập (markdown thuần). `Bai_Giang_Data_Governance.html` là bài giảng nhập môn dạng trang đơn (có dark mode toggle riêng). `DG_Checklist_DE_DA.html` là checklist vận hành cho Data Engineer (Airflow ingest → dbt transform → semantic layer) và Data Analyst (nhận đầu bài business → Power BI modeling), theo phong cách ADS, tick chọn lưu qua `localStorage`. Cả 2 file `.html` có bộ chọn nhanh liên kết sang tài liệu khác. | Mở thẳng file `.html`/`.md` tương ứng bằng trình duyệt/editor. |
| `RAG/` | REST API RAG (Retrieval-Augmented Generation) tra cứu tri thức về chính các dự án con trong repo này (dùng cho chatbot ngoài như Dify/N8N gọi vào) — xem `_REQUIREMENT.md` để biết kiến trúc chi tiết. Dùng Gemini (embedding + rerank + generation), Chroma làm vector store cục bộ (`index/`, đã gitignore vì generate lại được), hybrid search (dense + BM25 qua Reciprocal Rank Fusion), multi-turn, incremental ingest theo hash nội dung. `corpus/` là 10 file markdown nạp vào (có cả nội dung `fabricated` cố ý để test retrieval, không phải tài liệu gốc dự án). `chat.html` là UI test nhanh. Cần `GEMINI_API_KEY` trong `RAG/.env` (đã gitignore, không commit). | `python ingest.py` (nạp/refresh corpus) rồi `uvicorn api:app --host 0.0.0.0 --port 8000` (chạy API); hoặc `python query.py` để hỏi-đáp nhanh qua CLI. |

## Quy ước chung

- Không có `package.json`/`requirements.txt` ở đâu trong repo — mỗi script tự chứa import riêng (pandas, sklearn, xgboost, lightgbm, shap, h3...). Nếu thiếu package, hỏi user trước khi cài, không tự ý `pip install`.
- Không có test suite hay formatter chính thức. Đừng đề xuất thêm CI/lint framework trừ khi được yêu cầu.
- File `.csv`/`.png`/`.pkl` trong các thư mục là **output đã sinh ra**, không phải input cần chỉnh sửa tay. Khi cần xem nội dung CSV lớn, dùng `pandas.read_csv(...).head()` thay vì đọc toàn bộ file.
- `presentation.html` ở mỗi thư mục thường là **file được generate tự động** bởi script Python tương ứng (vd `build_presentation.py`) — sửa trực tiếp trong HTML sẽ bị ghi đè lần chạy sau; nên sửa ở script sinh ra nó.
- **File mô tả đề bài/yêu cầu/kế hoạch của mỗi thư mục con luôn đặt tên `_REQUIREMENT.md`** (dấu gạch dưới ở đầu để file luôn nổi lên đầu danh sách, dễ tìm khi mở thư mục). Khi review một thư mục con và không rõ đề bài gốc, tái dựng lại yêu cầu từ code/output có sẵn rồi ghi ra `_REQUIREMENT.md` theo đúng quy tắc đặt tên này — không đặt tên khác (vd `REQUIREMENT.md`, `requirements.md`).
- **Xây/redesign trang HTML kiểu report/dashboard/checklist**: dùng skill `design-html-page` (`.claude/skills/design-html-page/SKILL.md`) — skill này tự chứa toàn bộ ngôn ngữ thiết kế Analytics Design System (màu, typography, spacing, rule full-width), không cần đọc file nào khác. `design_key_visual/design_language_spec.md` là bản gốc/tham khảo hình ảnh mẫu, nội dung đã được sao chép đủ vào skill nên có thể bị xoá bất cứ lúc nào mà không ảnh hưởng.
- **Xây/redesign report/dashboard trong Power BI thật** (qua `powerbi-report-mcp`, PBIR): dùng skill `design-powerbi-report` (`.claude/skills/design-powerbi-report/SKILL.md`) — ngôn ngữ thiết kế dashboard vận hành kiểu "Merchant Hub / OMD DATA" (ban đầu rút ra từ 1 file PDF mẫu, nay đã xoá; skill tự chứa toàn bộ nội dung, không phụ thuộc file đó nữa). Khác với `design-html-page` (trang HTML trình bày, nhiều whitespace) — skill này dành cho report Power BI mật độ dày, nhiều widget.
- **7 file tài liệu tham khảo** (`Basic DA/DA_Interview_Lecture.html`, 3 file `Basic DE/*.html`, `Data Governance/DG_Checklist_DE_DA.html`, `Data Governance/Bai_Giang_Data_Governance.html`, `Basic ML/presentation.html`) đều theo phong cách ADS và có **bộ chọn nhanh** (dropdown, dính đầu trang) để nhảy qua lại giữa 7 file này. Khi thêm/đổi tên một trong 7 file, phải cập nhật lại đủ cả 7 nơi (danh sách `<option>` giống hệt nhau ở mỗi file, chỉ khác option nào được đánh dấu `selected disabled`).
- Ngôn ngữ giao tiếp mặc định trong repo này: **tiếng Việt**.
- Khi xưng hô, luôn tự gọi mình là **"bé"** (ngoan ngoãn, dễ thương) thay vì "tôi"/"mình" — nhưng vẫn trả lời ngắn gọn, không dài dòng.
